# Tangible Linear

Apply **Tangible Careers'** Linear conventions whenever you create, update, or organize
work in Linear (via the Linear MCP). Full rules: `skills/tangible-linear/SKILL.md` and
`skills/tangible-linear/guidelines.md`.

## Rules that matter (agents get these wrong)

- **Sprints are `sprint/YYYY-MM-DD` LABELS** (the sprint's Monday, ISO) — **NOT** Linear Cycles.
- **Label taxonomy only** (lowercase): `business-req`, `engineering-goal`, `ai-eval`,
  `backend`, `frontend`, `tech-debt`, `blocked`, `docs`, `sprint/YYYY-MM-DD`. Don't invent labels.
- **Status hygiene:** move an issue Todo → In Progress when you pick it up, → Done when
  finished. Never do the work then batch-jump to Done.
- **Dependencies:** `blocked by` relation **+** the `blocked` label. Incidental cross-effect
  → a `related` link (don't merge projects).
- **Project** only if it can END; forever-buckets are labeled backlog issues, not projects.

## Triggers (apply the conventions automatically)

- "file/create an issue", "add to Linear", "update the issue" → create/update per conventions
- "what's on my plate", "gather my work for sprint planning" → the 3-bucket work dump
- "plan the sprint", "commit these to the sprint" → the `sprint/YYYY-MM-DD` label workflow
